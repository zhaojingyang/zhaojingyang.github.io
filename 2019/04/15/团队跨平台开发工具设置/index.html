团队跨平台开发工具设置

发表于 2019-04-15 | 分类于 规范 ， Tools | | 阅读次数: |
字数统计: 714 字 | 阅读时长 ≈ 3 分钟

痛点

现阶段软件开发中大家可能都会使用到git或者ide。可能都会有跨平台的场景，团队人员的系统不一致，有的人使用Windows电脑，有的人使用linux，mac。或者大家都使用同一个操作系统的电脑，但是代码管理平台和开发系统不一致。这样就有可能由于 换行 带来问题。比如idea中pull下代码后，diff代码时由于换行而不能显示，只能通过show history来查看。

关于换行

在各操作系统下，文本文件所使用的换行符是不一样的。UNIX/Linux 使用的是 0x0A（LF），早期的 Mac OS 使用的是 0x0D（CR），后来的 OS X 在更换内核后与 UNIX 保持一致了。但 DOS/Windows 一直使用 0x0D0A（CRLF）作为换行符。

git 中换行的控制与设置

  • core.autocrlf

    (1) input

    If you’re on a Windows machine, set it to true — this converts LF endings into CRLF when you check out code:
    $ git config --global core.autocrlf true
    如果是在 Windows 系统上，把它设置成 true，这样在检出代码时，换行(LF)会被转换成回车和换行(CRLF)

    (2) input

    If you’re on a Linux or Mac system that uses LF line endings, then you don’t want Git to automatically convert them when you check out files; however, if a file with CRLF endings accidentally gets introduced, then you may want Git to fix it. You can tell Git to convert CRLF to LF on commit but not the other way around by setting core.autocrlf to input:
    $ git config --global core.autocrlf input
    如果使用以换行作为行结束符的 Linux 或 Mac，你不需要 Git 在检出文件时进行自动的转换；然而当一个以回车加换行作为行结束符的文件不小心被引入时，你肯定想让 Git 修正。 你可以把 core.autocrlf 设置成 input 来告诉 Git 在提交时把回车和换行转(CRLF)换成换行(LF)，检出时不转换

    (3) false

    If you’re a Windows programmer doing a Windows-only project, then you can turn off this functionality, recording the carriage returns in the repository by setting the config value to false:
    $ git config --global core.autocrlf false
    如果你是 Windows 程序员，且正在开发仅运行在 Windows 上的项目，可以设置 false 取消此功能，把回车(CR)保留在版本库中`

  • core.safecrlf

    • false 不做任何检查
    • warn 在提交时候检查并警告
    • true 在提交时检查，如果发现混用则拒绝提交
  • 建议的配置
    • Windows下
      core.autocrlf 设置为true
    • Mac/Linux
      core.autocrlf 设置为false
    • core.safecrlf 设置为true或warn

idea中设置换行

idea中大家都设置成统一的换行方式，eclipse中也有相关设置本文省略

  1. File->Settings 或工具栏的扳手(views-toolbar开启)
  2. Editor->Code Style
  3. 设置Line separator(for new files) Unix and OSX (\n)
